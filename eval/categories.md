# Категории вопросов для экспериментальной оценки

Используется пять категорий. Они нужны для последующего анализа ошибок по типу пользовательской формулировки.

| Категория | Назначение | Ожидаемое поведение поиска |
|---|---|---|
| `term_exact` | Вопрос содержит точный юридический термин или название правового института. | Полнотекстовый поиск должен показывать высокий результат; векторный поиск служит контролем. |
| `paraphrase` | Вопрос задан бытовым языком без точного совпадения с формулировкой закона. | Векторный и гибридный поиск должны иметь преимущество над чистым полнотекстовым поиском. |
| `mixed` | Вопрос содержит и бытовые слова, и юридические термины. | Ожидается устойчивость гибридного поиска. |
| `multi_fragment` | Для полного ответа нужны две или более статьи либо несколько фрагментов одной статьи. | Оценивается не только попадание одного фрагмента, но и Recall@K по нескольким эталонам. |
| `clarification_needed` | Вопрос сформулирован недостаточно конкретно или зависит от условий случая. | Извлечение должно найти общие нормы, а генерация должна явно указать на необходимость уточнений. |

Поле `needs_clarification_expected` относится к ручной проверке генерации ответа, а не к автоматической оценке поиска. Для оценки извлечения используются только записи, у которых после разрешения есть непустой `gold_chunk_ids`.
